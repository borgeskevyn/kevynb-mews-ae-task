# `fct_pnl_monthly` — Data Exploration

Ad hoc exploration of the `fct_pnl_monthly` fact table after widening its grain to
`(entity, territory, business_unit, consolidation_group, ix_code, salesforce_customer, month)`.
Queried directly against the local Postgres instance (mock data, generated by
`src/build_mock.py`) after running `dbt build`. All EUR figures are converted using
`stg_fx_rates`.

To reproduce: `cd dbt && DBT_PROFILES_DIR=. uv run dbt build`, then query
`fct_pnl_monthly` / `int_ledger_posted_eur` with psql or your tool of choice.

The 92.7% figure confirms the widened grain (6 dimension keys + month) lands very

so this mart is now closer to a granular star-schema fact than a heavy rollup.

## Open questions for follow-up

- **ENT3's margin gap** — its weaker margin shows up across territories rather than
  in one outlier; worth checking which accounts/business units within ENT3 drive the
  higher expense ratio.
- **June's dip** — net margin fell from 57.6% (March) to 43.8% (June) on stable
  volume. Worth checking whether expense mix (e.g. more adjustment/manual entries)
  shifted in June.
- **US/CZ margins** — both territories sit ~10pp below the territory average; a
  business-unit or account-type breakdown within those territories would clarify
  whether it's cost structure or a few large expense lines.
